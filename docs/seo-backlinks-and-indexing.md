# SEO Backlinks and Indexing Plan

## Search Console setup
### Google Search Console
1. Go to https://search.google.com/search-console
2. Add the property `https://berinkarjala.netlify.app`
3. Verify ownership (recommended: DNS verification through your domain provider)
4. Submit the sitemap: `https://berinkarjala.netlify.app/sitemap.xml`
5. Use URL Inspection to request indexing for:
   - `/`
   - `/about`
   - `/project`
   - `/projects/Minesweeper`
   - `/projects/SudokuSolver`

### Bing Webmaster Tools
1. Go to https://www.bing.com/webmasters/
2. Add the same site property
3. Verify via DNS or Search Console import
4. Submit the sitemap: `https://berinkarjala.netlify.app/sitemap.xml`
5. Request indexing for key URLs using URL Submission

## Backlink actions that are safe and practical
1. Update LinkedIn profile
   - Add the portfolio URL in the Featured section
   - Add the portfolio URL to the Contact info section
2. Update GitHub profile
   - Add the portfolio URL in the profile bio and pinned project list
3. Project write-ups
   - Write brief case studies for Minesweeper and Sudoku Solver
   - Post to dev.to, Hashnode, or Medium and link back to the portfolio
4. Community directories
   - Add the portfolio to relevant developer directories
   - Examples: Hashnode community, dev.to profiles, Indie Hackers profile
5. Cross-link between properties
   - GitHub README for portfolio repo links back to the live site
   - Any personal resume or PDF includes the portfolio URL

## Indexing and crawl hygiene
- Keep the sitemap updated when new routes are added.
- Avoid duplicate routes and canonical mismatches.
- Ensure all pages have unique titles and descriptions.

## Monthly SEO maintenance checklist
- Run Lighthouse on the homepage and two interior pages.
- Recheck Search Console coverage and fix crawl errors.
- Inspect top queries for “Berin Karjala” and adjust copy if needed.
- Validate structured data with Google Rich Results Test.
- Refresh project descriptions and add one new backlink.
